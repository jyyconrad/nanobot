# Writing Skill

写作技能 - 文档编写和内容创作。

## 何时使用此技能

当任务涉及以下内容时使用：
- 编写 API 文档
- 创建用户手册
- 撰写 README 文件
- 编写技术博客文章
- 文档翻译

## 写作原则

### 1. 清晰性

- 使用简单、直接的语言
- 避免复杂的句子结构
- 保持段落简洁（3-5句）
- 使用主动语态

### 2. 简洁性

- 删除不必要的词汇
- 避免重复
- 使用精确的描述
- 保持内容紧凑

### 3. 结构化

- 使用标题和子标题
- 使用列表和表格
- 保持逻辑顺序
- 提供清晰的导航

### 4. 一致性

- 使用一致的术语
- 保持相同的写作风格
- 使用统一的格式
- 遵循文档风格指南

## 写作技巧

### 1. 受众分析

- 了解读者的背景和需求
- 使用适当的技术术语
- 提供必要的背景信息

### 2. 内容组织

- 从一般到具体
- 重要信息优先
- 使用过渡词连接段落
- 保持逻辑顺序

### 3. 语言使用

- 使用主动语态
- 避免被动语态
- 使用精确的动词
- 避免模棱两可的词汇

### 4. 格式和排版

- 使用适当的字体和大小
- 保持一致的间距
- 使用高亮和粗体强调重点
- 保持适当的行高

## 常见写作问题

### 1. 内容冗长

**症状**：段落过长，难以阅读。

**解决方案**：
- 分解长段落
- 使用列表
- 删除不必要的词汇
- 保持段落专注于单个主题

### 2. 术语不一致

**症状**：同一概念使用不同术语。

**解决方案**：
- 创建术语表
- 使用一致的词汇
- 定期审查文档
- 与团队沟通术语

### 3. 缺乏实例

**症状**：文档难以理解，没有示例。

**解决方案**：
- 添加代码示例
- 提供截图
- 包含详细的步骤说明
- 使用实际案例
