# 架构决策记录 (ADR)

本目录包含 Nanobot 系统的所有架构决策记录 (Architecture Decision Records)。

## 什么是 ADR？

架构决策记录 (ADR) 是一个文档，它描述了一个影响软件系统架构的重要决策，以及该决策背后的背景和后果。

## ADR 格式

每个 ADR 遵循以下结构：

```markdown
# [编号]. [标题]

日期: [YYYY-MM-DD]
状态: [提议/接受/已弃用/已取代]

## 背景

描述决策背后的上下文和问题。

## 决策

描述所做的决策。

## 后果

描述该决策的正面和负面后果。

## 替代方案

描述被考虑的替代方案以及为什么它们被拒绝。
```

## 创建新的 ADR

要创建新的 ADR：

1. 复制 `template.md` 文件
2. 使用下一个编号重命名（例如：`005-决策名称.md`）
3. 填写所有部分
4. 提交 PR 进行审查
