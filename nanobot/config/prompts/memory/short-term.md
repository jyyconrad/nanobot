# 短期记忆策略

## 记忆触发条件

### 1. 任务相关信息
- 用户明确要求记住的任务细节（如截止时间、优先级、关键步骤）
- 正在进行中的任务上下文（如当前处理阶段、遇到的问题、临时解决方案）
- 任务相关的临时数据（如计算结果、中间变量、临时文件路径）

### 2. 交互上下文
- 用户的提问意图和需求背景
- 对话中的关键转折点和决策点
- 未完成的待办事项或后续行动项

### 3. 环境状态
- 当前工作目录和文件路径
- 正在执行的命令或程序状态
- 系统资源使用情况（如内存、CPU 占用）

### 4. 学习和探索过程
- 正在研究的技术文档和教程内容
- 实验过程中的参数设置和结果
- 调试过程中的错误信息和修复尝试

## 记忆保存格式

### 1. 任务上下文记录
```markdown
# 任务: [任务名称]
- 状态: [进行中/暂停/待开始]
- 截止时间: [YYYY-MM-DD HH:MM]
- 优先级: [高/中/低]
- 关键步骤:
  - [步骤1]
  - [步骤2]
  - [步骤3]
- 当前进度: [已完成x% / 正在处理步骤x]
- 遇到的问题: [问题描述]
- 临时解决方案: [解决方案]
```

### 2. 交互历史记录
```markdown
# 对话: [用户问题摘要]
- 时间: [YYYY-MM-DD HH:MM:SS]
- 用户意图: [主要需求]
- 关键信息: [核心数据]
- 待办事项: [后续行动]
- 参考资源: [链接/文档]
```

### 3. 环境状态记录
```markdown
# 环境状态
- 时间: [YYYY-MM-DD HH:MM:SS]
- 工作目录: [路径]
- 正在执行的命令: [命令]
- 系统资源:
  - 内存: [使用量/总量]
  - CPU: [使用率]
  - 磁盘: [可用空间]
- 网络状态: [连接/断开]
```

### 4. 学习过程记录
```markdown
# 学习主题: [技术名称]
- 时间: [YYYY-MM-DD HH:MM:SS]
- 学习资源: [文档/教程链接]
- 关键概念: [核心知识点]
- 实验参数: [配置信息]
- 实验结果: [输出数据]
- 问题与解决方案: [错误信息+修复方法]
```

## 记忆有效期

- 任务相关记忆: 任务完成后保留 24 小时
- 交互上下文记忆: 对话结束后保留 12 小时
- 环境状态记忆: 状态变化后保留 6 小时
- 学习过程记忆: 学习结束后保留 48 小时

## 记忆更新策略

- 任务状态变更时立即更新
- 对话内容新增时实时记录
- 环境状态变化时自动检测并更新
- 学习过程中定期（每 15 分钟）保存快照
