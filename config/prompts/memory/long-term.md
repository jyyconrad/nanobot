# 长期记忆策略

## 记忆触发条件

### 1. 任务完成总结
- 任务最终解决方案和实现细节
- 任务执行过程中的关键经验和教训
- 任务成果和交付物清单

### 2. 技术知识积累
- 反复使用的代码片段和工具方法
- 技术架构设计和最佳实践
- 常见问题的解决方案和调试技巧

### 3. 用户偏好和习惯
- 用户的编码风格和偏好
- 常用工具和工作流程
- 沟通方式和反馈习惯

### 4. 项目和业务上下文
- 项目架构和代码库结构
- 业务领域知识和术语
- 关键业务流程和规则

### 5. 学习成果固化
- 系统学习的技术知识体系
- 解决复杂问题的方法论
- 个人技能提升的记录

## 记忆保存格式

### 1. 任务完成总结
```markdown
# 任务: [任务名称] - 完成总结
- 完成时间: [YYYY-MM-DD HH:MM]
- 最终解决方案: [核心实现思路]
- 关键代码片段: [代码示例]
- 经验教训: [成功经验/失败原因]
- 优化建议: [改进方案]
- 交付物: [文件/功能清单]
```

### 2. 技术知识卡片
```markdown
# 技术卡片: [知识点名称]
- 分类: [技术领域/应用场景]
- 核心概念: [简洁定义]
- 实现方法: [代码示例/步骤]
- 适用场景: [使用条件]
- 常见问题: [错误类型+解决方案]
- 参考资源: [文档/链接]
```

### 3. 用户信息档案
```markdown
# 用户档案: [用户名]
- 编码风格: [代码规范/偏好]
- 常用工具: [编辑器/IDE/插件]
- 工作流程: [开发/测试/部署流程]
- 沟通风格: [反馈方式/语言习惯]
- 专业领域: [技术栈/业务熟悉度]
- 学习偏好: [学习方式/资源类型]
```

### 4. 项目知识库
```markdown
# 项目: [项目名称]
- 架构: [技术栈/系统设计]
- 代码结构: [目录组织/模块划分]
- 业务领域: [行业/功能描述]
- 核心流程: [主要业务流程]
- 关键模块: [功能模块清单]
- 部署方式: [环境配置/发布流程]
```

### 5. 学习成果记录
```markdown
# 学习成果: [学习主题]
- 学习时间: [开始-结束时间]
- 知识体系: [主要知识点框架]
- 关键技能: [掌握的核心能力]
- 实践案例: [应用场景/项目]
- 学习资源: [课程/文档/书籍]
- 能力评估: [掌握程度/水平]
```

## 记忆存储策略

- **分层存储**: 按主题、技术领域、项目分类存储
- **标签系统**: 为每个记忆条目添加关键词标签
- **索引机制**: 建立快速检索的索引系统
- **版本管理**: 对更新的知识进行版本控制

## 记忆更新策略

- **定期回顾**: 每月对长期记忆进行一次梳理和更新
- **增量学习**: 新知识点与现有体系关联存储
- **过时淘汰**: 定期清理不再适用的知识内容
- **质量评估**: 对记忆内容进行准确性和实用性评估

## 记忆检索策略

- **关键词搜索**: 支持多维度关键词组合搜索
- **语义匹配**: 基于内容语义的智能匹配
- **关联推荐**: 根据当前上下文推荐相关记忆
- **分类浏览**: 按技术领域、项目、主题分类浏览
